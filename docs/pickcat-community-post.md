# Pickcat 复活计划：一个消失的编程猫移动社区，为什么还值得被找回来

大家好，这里想认真记录一个已经做了很久、也越来越不像“玩票”的项目。

它叫 **Pickcat Reborn**。

这不是一个全新的 App 点子，而是一次**尽量尊重历史资料、界面截图、残留接口和原有使用习惯**的复刻工程。目标很简单：

> 把已经消失的 Pickcat，重新做成一个还能跑、还能看、还能继续讲故事的版本。

---

## 1. Pickcat 是什么

如果你接触过编程猫社区，大概会知道社区长期以网页为主，也有过很多不同阶段、不同形态的移动端尝试。

而 Pickcat，可以理解成编程猫社区在手机端的一次更“社区化”的形态探索：

- 它有独立的首页推荐流
- 有喵圈/板块入口
- 有消息流
- 有个人主页
- 有作品浏览页
- 有更偏移动端的交互和视觉包装

从目前还能找到的历史截图、旧帖子、残留 API、旧 APK 资源来看，Pickcat 并不是简单把网页包一层壳，而是有自己一套偏移动端的页面组织方式。

---

## 2. 它为什么会“没了”

这个项目现在已经无法直接使用，主要有几个原因：

1. 原始 APK 已经无法正常复活
2. APK 存在加固，静态逆向很难直接拿到真实业务代码
3. 原服务端和部分旧接口状态已经变化
4. 原来的分发和运行环境也已经断代

也就是说，现在要做的不是“修一下旧包”，而是：

**从现有资料里，把它重新拼回来。**

---

## 3. 这次复活项目到底在做什么

这次工程不是“魔改旧 APK”，而是三条线一起推进：

### 资料线

- 提取旧 APK 的 `assets / res / 图标 / 背景 / 登录图 / 底部导航素材`
- 搜索社区残留帖子、旧截图、论坛讨论
- 整理历史界面特征

### 接口线

- 分析编程猫社区仍可访问的接口
- 接入论坛、作品、用户、消息等可用数据
- 用尽量接近旧体验的方式重新组织这些数据

### 体验线

- 做一个能跑的前端原型
- 尽量贴近 Pickcat 的移动端布局和节奏
- 在保留“像 Pickcat”的前提下补足今天还能工作的逻辑

## 5. 现在已经做到什么程度

当前版本已经实现了这些能力：

### 首页

- 推荐 / 发现 / 关注 三个主 feed
- 推荐页接论坛帖子流
- 发现页接作品流
- 关注页开始接关注者清单与动态近似流

### 喵圈

- 接入论坛板块
- 支持进入板块
- 支持板块分页加载更多帖子

### 作品页

- 能打开作品详情
- 能识别不同引擎类型
- 已补 `Nemo / Kitten / KittenN / 海龟` 的播放器入口策略
- 已补作品点赞、收藏、评论、回复、作者关注
- 已补“TA 的更多作品”和“推荐作品”

### 用户页

- 点击用户名可进入用户详情
- 显示作品、粉丝、关注、获赞等指标
- 已支持粉丝与关注用户列表浏览

### 消息页

- 已接多种消息类型
- 点击消息卡片能跳到对应帖子或作品
- 会尽量定位到原评论/原回复附近

### 我的页面

- 显示个人资料
- 显示作品列表
- 补了粉丝和关注信息读取

## 7. 这个项目里最难的部分是什么

最难的不是把界面画出来，而是下面这几件事：

### 7.1 旧 APK 不能直接当源码用

原包存在加固，动态运行也不稳定，没法像普通 Android 项目那样直接改。

### 7.2 截图是真实参考，但截图不会告诉你数据怎么流

很多东西只能靠：

- 旧截图
- 板块帖子
- 社区残留接口
- 资源文件命名
- 页面结构推测

一层一层拼回去。

### 7.3 今天的社区接口，不等于当年的 Pickcat 接口

现在能用的数据接口，很多属于现存社区，而不是当年 Pickcat 完整的后端。

所以复刻过程中要做很多“近似映射”：

- 找能用的真实数据
- 再组织成更像 Pickcat 的形式

## 8. 现在项目的状态

如果用一句话概括现在的状态：

> 它已经不是“只能看看截图”的阶段了，而是一个可运行、可打包、可继续补完的 Pickcat 复活原型。

目前已有：

- Web 原型
- Windows 运行产物
- Android 调试版产物
- 本地整理好的截图索引
- 持续补全中的接口映射

但它还没有到“完全一比一还原”的程度。

离真正成熟还差：

- 更多历史截图
- 更完整的首页布局还原
- 更细的消息与社交流程
- 更多编辑器作品的兼容

---

## 9. 这项目接下来还会继续做什么

下一阶段我最想继续推进的方向：

1. 继续搜更多 Pickcat 历史截图
2. 把首页、发布页做得更贴近原版
3. 把用户页、消息页、作品页继续补齐
4. 把不同编辑器作品的 player 适配做得更稳
5. 尽量让这个项目不只是“怀旧展示”，而是真能继续用

## 10. 最后

很多老项目消失以后，最后留下的只有：

- 几张截图
- 一点模糊记忆
- 零散帖子

但 Pickcat 这个项目最打动我的地方就在于：

**它并不是完全没有痕迹。**

它还留下了资源、界面、使用痕迹、社区讨论、历史帖子和接口线索。

所以这次复活，本质上不是“空想一个旧产品”，而是：

**尽可能把一段已经中断的社区体验，重新接回今天。**
